# ExpandableGroup Component

The ExpandableGroup component organizes multiple Expandable components into a cohesive group, providing consistent styling and behavior for related collapsible sections.

## Overview

Use ExpandableGroup components to create FAQ sections, troubleshooting guides, or any collection of related expandable content that benefits from unified presentation and interaction patterns.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `children` | `Expandable[]` | Yes | Array of Expandable components |
| `allowMultiple` | `boolean` | No | Whether multiple sections can be open simultaneously |
| `defaultOpenIndex` | `number` | No | Index of the section to open by default |
| `className` | `string` | No | Additional CSS classes for styling |

## Behavior Modes

### Accordion Mode (`allowMultiple={false}`)
- Only one section can be open at a time
- Opening a new section closes the previously open one
- Good for FAQs and step-by-step guides

### Multi-Expand Mode (`allowMultiple={true}`)
- Multiple sections can be open simultaneously
- Each section operates independently
- Good for reference documentation and troubleshooting

## Accessibility

- Maintains proper focus management across the group
- Uses ARIA attributes for group relationships
- Supports keyboard navigation between sections
- Provides clear state announcements for screen readers

## Best Practices

- Group logically related expandable content
- Use consistent title formatting within a group
- Consider the appropriate behavior mode for your use case
- Limit the number of items in a group (5-10 recommended)
- Provide clear, descriptive titles for each section
