# Table Component

The Table component provides structured data display with support for headers, sorting, and responsive design, perfect for presenting tabular information in documentation.

## Overview

Use Table components to display structured data, comparison charts, feature matrices, API reference information, and other tabular content that benefits from organized rows and columns.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `headers` | `string[]` | Yes | Array of column header names |
| `rows` | `string[][]` | Yes | Array of row data (each row is an array of cell values) |
| `caption` | `string` | No | Table caption for accessibility |
| `sortable` | `boolean` | No | Enable column sorting |
| `striped` | `boolean` | No | Alternate row background colors |
| `bordered` | `boolean` | No | Add borders around cells |
| `compact` | `boolean` | No | Reduce padding for denser display |
| `className` | `string` | No | Additional CSS classes for styling |

## Usage Examples

### Basic Table

```mdx
<Table 
  headers={["Name", "Type", "Required", "Description"]}
  rows={[

    ["email", "string", "No", "Contact email"]
  ]}
/>
```

## Responsive Design

Tables automatically adapt to smaller screens:

- **Horizontal Scrolling**: Wide tables scroll horizontally on mobile
- **Stacked Layout**: Option to stack cells vertically on narrow screens
- **Priority Columns**: Important columns remain visible on mobile
- **Compact Mode**: Reduced padding for better mobile display

## Accessibility Features

- **Semantic Markup**: Uses proper table elements (table, thead, tbody, th, td)
- **Caption Support**: Table captions for screen reader context
- **Header Association**: Column headers properly associated with data cells
- **Keyboard Navigation**: Full keyboard support for interactive tables
- **Sort Announcements**: Screen reader announcements for sort changes

## Sorting Functionality

When `sortable={true}`:

- Click column headers to sort by that column
- First click sorts ascending, second click sorts descending
- Visual indicators show current sort column and direction
- Keyboard accessible with Enter and Space keys
- Maintains accessibility with proper ARIA attributes

## Performance Considerations

- **Virtual Scrolling**: Large tables use virtualization for performance
- **Lazy Loading**: Table data can be loaded progressively
- **Memoization**: Prevents unnecessary re-renders of table content
- **Efficient Sorting**: Client-side sorting optimized for large datasets

## Best Practices

- Always provide meaningful column headers
- Use table captions for complex tables
- Keep cell content concise and scannable
- Consider responsive design for mobile users
- Use consistent data formatting within columns
- Provide sorting for large datasets
- Test keyboard navigation and screen reader compatibility

## Styling Options

### Striped Rows
Alternating background colors improve readability for wide tables.

### Bordered Cells
Borders help separate data in dense tables with many columns.

### Compact Mode
Reduced padding fits more data in limited space.
