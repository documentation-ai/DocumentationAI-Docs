## Overview

The `<Update>` component creates visually structured changelog entries and product updates with a sticky label sidebar and dedicated content area. Perfect for documenting releases, feature announcements, and product evolution over time.

Common use cases:
- Changelog entries with version numbers and release dates
- Product update announcements with timestamps
- Feature release timelines and roadmaps
- API version deprecation notices
- Release notes documentation
- Monthly or weekly update summaries

````jsx
<Update label="2024-10-11" description="v0.1.0">
  ## New features

  We've added support for real-time collaboration and team workspaces.
  
  - Multi-user editing with live cursor tracking
  - Team workspace management
  - Enhanced permission controls
  - Activity feed and notifications
</Update>
````

## Basic syntax

Use the `<Update>` component to create changelog entries with labels and descriptions:

````jsx
<Update label="2024-10-15" description="v1.2.0">
  Released new analytics dashboard with real-time metrics and custom reporting.
</Update>
````

<Update label="2024-10-15" description="v1.2.0">
  Released new analytics dashboard with real-time metrics and custom reporting.
</Update>

<Callout kind="info">
  The label sidebar uses sticky positioning and remains visible as users scroll through long update content.
</Callout>

## Version releases

Document software releases with version numbers and detailed changes:

````jsx
<Update label="2024-10-20" description="v2.0.0">
  ## Major release
  
  This release includes breaking changes and new features.
  
  **Breaking changes:**
  - Updated API authentication flow requiring new credentials
  - Removed deprecated webhook endpoints
  
  **New features:**
  - Advanced search with filters and operators
  - Export functionality for reports and data
  - Dark mode support across all interfaces
  
  **Bug fixes:**
  - Fixed race condition in real-time sync
  - Resolved memory leak in large document rendering
</Update>
````

## Code examples in updates

Include code snippets to demonstrate changes and new features:

````jsx
<Update label="2024-10-05" description="API Enhancement">
  ## New authentication method
  
  We've added support for OAuth 2.0 authentication alongside existing API keys.
  
  <CodeGroup tabs="JavaScript,Python,cURL">
  ```javascript
  const response = await fetch('https://api.example.com/data', {
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  });
  ```
  
  ```python
  import requests
  
  response = requests.get(
    'https://api.example.com/data',
    headers={'Authorization': f'Bearer {access_token}'}
  )
  ```
  
  ```bash
  curl -H "Authorization: Bearer YOUR_TOKEN" \
    https://api.example.com/data
  ```
  </CodeGroup>
  
  <Callout kind="alert">
    API key authentication will be deprecated in version 3.0. Migrate to OAuth 2.0 before December 2024.
  </Callout>
</Update>
````

## Attributes

<ParamField path="label" param-type="string" required="true" showLocation="false">
  The primary label displayed in the sidebar. Typically used for dates (YYYY-MM-DD format), version identifiers, or update categories.
</ParamField>
<ParamField path="description" param-type="string" required="true" showLocation="false">
  Supporting description displayed below the label. Commonly used for version numbers, release types, or brief categorizations.
</ParamField>
<ParamField path="children" param-type="node" required="true" showLocation="false">
  The main content of the update. Supports all MDX content including headings, text, code blocks, images, cards, callouts, and other components.
</ParamField>

## Layout behavior

The Update component uses a responsive grid layout:

- **Desktop (lg breakpoint and above)**: Four-column grid with label sidebar taking one column and content spanning three columns
- **Mobile and tablet**: Stacks vertically with label section above content
- **Sidebar positioning**: Label section uses sticky positioning (top: 1.5rem) on desktop to remain visible during scroll
- **Spacing**: Vertical padding of 2rem (py-8) separates updates when stacked

<Callout kind="info">
  The label sidebar automatically adjusts from sticky to static positioning on mobile devices for optimal readability.
</Callout>

## Common patterns

- **Release changelogs** - Document software versions with labels as dates and descriptions as version numbers, organizing features, fixes, and breaking changes.
- **Product roadmap updates** - Share development progress with labels indicating quarters or months and descriptions showing milestone names.
- **Feature announcements** - Highlight new capabilities with launch dates as labels and feature categories as descriptions, including demos and documentation links.
- **API versioning** - Communicate API changes with version numbers as descriptions and release dates as labels, providing migration guides and code examples.
- **Security advisories** - Publish security updates with CVE identifiers or severity levels in descriptions and disclosure dates as labels.
- **Maintenance notifications** - Announce scheduled maintenance windows with dates as labels and service impact descriptions.
- **Beta program updates** - Track beta feature evolution with phase indicators (Alpha, Beta, GA) and release dates for early adopter programs.
- **Integration launches** - Document new third-party integrations with partner names and integration capabilities in rich content sections.

<Callout kind="success">
  Combine Update components with other MDX components to create comprehensive, visually engaging changelog and announcement pages.
</Callout>
