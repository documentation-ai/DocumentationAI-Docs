# Callout Component

The Callout component creates visually distinct panels to highlight important information, warnings, tips, or notes within your documentation.

## Overview

Use Callout components to draw attention to critical information, provide helpful tips, or warn users about potential issues.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `kind` | `'note' \| 'tip' \| 'warning' \| 'danger' \| 'info'` | Yes | The type of callout |
| `children` | `ReactNode` | Yes | The content to display |
| `title` | `string` | No | Optional custom title for the callout |
| `icon` | `ReactNode` | No | Custom icon to display |
| `className` | `string` | No | Additional CSS classes for styling |

## Usage Examples

### Basic Callouts

```mdx
<Callout kind="note">
This is an important note that provides additional context.
</Callout>

<Callout kind="tip">
Here's a helpful tip to improve your workflow.
</Callout>

<Callout kind="warning">
Be careful when making this change as it may affect existing functionality.
</Callout>

<Callout kind="danger">
This action cannot be undone. Make sure you have a backup.
</Callout>
```

### With Custom Title

```mdx
<Callout kind="info" title="API Rate Limits">
The API has a rate limit of 1000 requests per hour per API key.
</Callout>
```

### With Rich Content

```mdx
<Callout kind="tip">
  <Paragraph>
    For better performance, consider using the following optimization:
  </Paragraph>
  <CodeBlock language="javascript">
  {`// Use memoization for expensive calculations
const memoizedResult = useMemo(() => {
  return expensiveCalculation(data);
}, [data]);`}
  </CodeBlock>
</Callout>
```

### Nested Content

```mdx
<Callout kind="warning">
  <Heading level={4}>Breaking Changes in v2.0</Heading>
  <List type="unordered">
    <li>API endpoints have changed</li>
    <li>Authentication method updated</li>
    <li>Response format modified</li>
  </List>
</Callout>
```

## Callout Types

### Note
- **Purpose**: General information and context
- **Color**: Blue
- **Icon**: Information icon
- **Use for**: Additional details, explanations, references

### Tip
- **Purpose**: Helpful suggestions and best practices
- **Color**: Green
- **Icon**: Lightbulb icon
- **Use for**: Optimization tips, shortcuts, recommendations

### Warning
- **Purpose**: Cautions and potential issues
- **Color**: Orange/Yellow
- **Icon**: Warning triangle
- **Use for**: Deprecated features, potential problems, careful considerations

### Danger
- **Purpose**: Critical warnings and destructive actions
- **Color**: Red
- **Icon**: Error/danger icon
- **Use for**: Irreversible actions, security risks, critical errors

### Info
- **Purpose**: Neutral informational content
- **Color**: Gray/Blue
- **Icon**: Info icon
- **Use for**: Status updates, general information, clarifications

## Accessibility

- Uses appropriate ARIA roles and labels
- Provides semantic meaning through color and icons
- Supports keyboard navigation and focus management
- Compatible with screen readers and assistive technologies

## Best Practices

- Use callouts sparingly to maintain their impact
- Choose the appropriate type for your message
- Keep content concise and actionable
- Don't nest callouts within other callouts
- Ensure color is not the only way to convey meaning.
