# Heading Component

The Heading component creates hierarchical section titles that provide structure and navigation landmarks for your documentation.

## Overview

Use Heading components to create a clear information hierarchy and enable users to quickly scan and navigate your content.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `level` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | Yes | The heading level (h1-h6) |
| `children` | `ReactNode` | Yes | The heading text content |
| `id` | `string` | No | Custom ID for anchor links |
| `className` | `string` | No | Additional CSS classes for styling |

## Usage Examples

### Basic Headings

```mdx
<Heading level={1}>Main Title</Heading>
<Heading level={2}>Section Title</Heading>
<Heading level={3}>Subsection Title</Heading>
```

### With Custom ID

```mdx
<Heading level={2} id="custom-section">
Custom Section
</Heading>
```

### Nested Structure

```mdx
<Heading level={1}>Getting Started</Heading>
<Heading level={2}>Installation</Heading>
<Heading level={3}>Prerequisites</Heading>
<Heading level={3}>Setup Steps</Heading>
<Heading level={2}>Configuration</Heading>
```

## Accessibility

- Uses semantic heading elements (h1-h6)
- Maintains proper heading hierarchy
- Provides navigation landmarks for screen readers
- Supports keyboard navigation and focus management

## Best Practices

- Use only one h1 per page
- Don't skip heading levels
- Keep headings concise and descriptive
- Use consistent heading styles throughout documentation
