# Request Component

The Request component provides a structured way to document HTTP API requests, including method, URL, headers, and request body examples.

## Overview

Use Request components to document API endpoints with complete request information, making it easy for developers to understand how to interact with your API.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `method` | `'GET' \| 'POST' \| 'PUT' \| 'PATCH' \| 'DELETE'` | Yes | HTTP method |
| `url` | `string` | Yes | The API endpoint URL |
| `headers` | `object` | No | Request headers object |
| `children` | `ReactNode` | No | Request body or additional content |
| `title` | `string` | No | Custom title for the request |
| `className` | `string` | No | Additional CSS classes for styling |

## Usage Examples

### GET Request

```mdx
<Request method="GET" url="/api/users">
  Retrieve a list of all users in the system.
</Request>
```

### POST Request with Body

```mdx
<Request method="POST" url="/api/users" title="Create New User">
  <CodeBlock language="json">
  {`{
    "name": "John Doe",
    "email": "john@example.com",
    "role": "user"
  }`}
  </CodeBlock>
</Request>
```

### Request with Headers

```mdx
<Request 
  method="PUT" 
  url="/api/users/{id}"
  headers={{
    "Authorization": "Bearer {token}",
    "Content-Type": "application/json"
  }}
>
  <CodeBlock language="json">
  {`{
    "name": "Jane Doe",
    "email": "jane@example.com"
  }`}
  </CodeBlock>
</Request>
```

### Complex API Request

```mdx
<Request 
  method="POST" 
  url="/api/projects/{projectId}/documents"
  headers={{
    "Authorization": "Bearer {token}",
    "Content-Type": "multipart/form-data"
  }}
  title="Upload Document"
>
  <Paragraph>
    Upload a new document to the specified project. Supports multiple file formats including PDF, DOCX, and Markdown.
  </Paragraph>
  
  <ParamField path="file" paramType="file" required={true}>
    The document file to upload (max 10MB).
  </ParamField>
  
  <ParamField path="title" paramType="string" required={true}>
    The title for the document.
  </ParamField>
  
  <ParamField path="description" paramType="string">
    Optional description of the document content.
  </ParamField>
  
  <ParamField path="tags" paramType="array">
    Array of tag names to associate with the document.
  </ParamField>
</Request>
```

### Query Parameters Example

```mdx
<Request method="GET" url="/api/search?q={query}&limit={limit}&offset={offset}">
  <Paragraph>
    Search for content across all documents with pagination support.
  </Paragraph>
  
  <Heading level={4}>Query Parameters</Heading>
  <ParamField path="q" paramType="string" required={true}>
    The search query string.
  </ParamField>
  
  <ParamField path="limit" paramType="number" defaultValue="20">
    Maximum number of results to return (1-100).
  </ParamField>
  
  <ParamField path="offset" paramType="number" defaultValue="0">
    Number of results to skip for pagination.
  </ParamField>
  
  <ParamField path="type" paramType="string">
    Filter results by content type: <code>document</code>, <code>image</code>, or <code>video</code>.
  </ParamField>
</Request>
```

### Authentication Request

```mdx
<Request method="POST" url="/auth/login" title="User Authentication">
  <Paragraph>
    Authenticate a user and receive an access token for API requests.
  </Paragraph>
  
  <CodeBlock language="json">
  {`{
    "email": "user@example.com",
    "password": "securepassword123",
    "remember": true
  }`}
  </CodeBlock>
  
  <Callout kind="note">
    The access token expires after 24 hours. Use the refresh token to obtain a new access token.
  </Callout>
</Request>
```

### Batch Request

```mdx
<Request method="POST" url="/api/users/batch" title="Create Multiple Users">
```

## HTTP Methods

### GET
- Used for retrieving data
- No request body typically
- Parameters passed via query string or URL path

### POST
- Used for creating new resources
- Request body contains the data to create
- Often returns the created resource

### PUT
- Used for updating entire resources
- Request body contains the complete updated resource
- Idempotent operation

### PATCH
- Used for partial updates
- Request body contains only the fields to update
- More efficient than PUT for small changes

### DELETE
- Used for removing resources
- May or may not have a request body
- Should be idempotent

## Accessibility

- Uses semantic markup for request information
- Provides clear method and URL identification
- Supports keyboard navigation and focus management
- Compatible with screen readers

## Best Practices

- Always specify the correct HTTP method
- Include all required headers in examples
- Provide realistic request body examples
- Document authentication requirements
- Include parameter validation rules
- Show both success and error scenarios
