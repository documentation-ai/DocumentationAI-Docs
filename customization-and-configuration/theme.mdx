Set up light and dark theme modes for your documentation site. The Documentation.AI platform automatically switches themes based on user preference or system settings while maintaining optimal contrast and accessibility.

Theme colors apply site-wide to all pages, but individual pages inherit these settings and can be customized further using [page-level front matter configuration](page-configuration).

## Prerequisites

- Access to your Documentation.AI project
- Basic understanding of hex color codes
- Your `documentation.json` file (located in your project root)

## Configure Dual Theme Support

Follow these steps to set up both light and dark themes for your documentation site.

### Step 1: Open your configuration file

Locate and open the `documentation.json` file in your project root directory.

### Step 2: Add the colors object

Add a `colors` object to your `documentation.json` file. If it already exists, you can modify the existing configuration.

```json
{
  "colors": {
    "light": {
      "brand": "#3143e3",
      "heading": "#1a1a1a", 
      "text": "#374151"
    },
    "dark": {
      "brand": "#85a1ff",
      "heading": "#f2f2f2",
      "text": "#c1c1c1"
    }
  }
}
```

### Step 3: Configure light theme colors

Set up colors for light mode by defining these properties under `colors.light`:

<ParamField path="brand" param-type="string" required="false" showLocation="false">
  Primary color for buttons, links, and accents. Use hex format (e.g., `#3143e3`).
</ParamField>

<ParamField path="heading" param-type="string" required="false" showLocation="false">
  Color for headings and titles. Default: `#1a1a1a`.
</ParamField>

<ParamField path="text" param-type="string" required="false" showLocation="false">
  Color for body text for optimal readability. Default: `#374151`.
</ParamField>

### Step 4: Configure dark theme colors

Set up colors for dark mode by defining these properties under `colors.dark`:

<ParamField path="brand" param-type="string" required="false" showLocation="false">
  Primary color optimized for dark backgrounds. Use hex format (e.g., `#85a1ff`).
</ParamField>

<ParamField path="heading" param-type="string" required="false" showLocation="false">
  Color for headings in dark mode. Default: `#f2f2f2`.
</ParamField>

<ParamField path="text" param-type="string" required="false" showLocation="false">
  Color for body text in dark mode for optimal readability. Default: `#c1c1c1`.
</ParamField>

### Step 5: Use proper hex format

Ensure all color values use hex format:
- Include the `#` symbol at the beginning
- Use 6-digit (`#3143e3`) or 3-digit (`#333`) hex codes
- Valid examples: `#3143e3`, `#fff`, `#1a1a1a`

### Step 6: Save and deploy

Save your `documentation.json` file and deploy your changes to see the theme configuration take effect.

## Verify Your Theme Setup

1. Visit your documentation site
2. Look for a theme toggle button (usually in the header)
3. Switch between light and dark modes
4. Confirm colors display correctly in both themes
5. Check that text remains readable in both modes
