# Iframe Component

The Iframe component enables embedding of external content, interactive demos, and third-party applications directly within your documentation pages.

## Overview

Use Iframe components to embed interactive demos, external tools, code playgrounds, maps, forms, and other web-based content that enhances your documentation with live examples.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `src` | `string` | Yes | URL of the content to embed |
| `width` | `number \| string` | No | Frame width (pixels or percentage) |
| `height` | `number \| string` | No | Frame height (pixels or percentage) |
| `title` | `string` | Yes | Descriptive title for accessibility |
| `allowFullscreen` | `boolean` | No | Allow fullscreen mode |
| `sandbox` | `string` | No | Sandbox restrictions for security |
| `loading` | `'lazy' \| 'eager'` | No | Loading behavior (default: lazy) |
| `className` | `string` | No | Additional CSS classes for styling |

## Security Considerations

### Sandbox Attribute

The `sandbox` attribute provides security restrictions:

- `allow-forms` - Allow form submission
- `allow-scripts` - Allow JavaScript execution
- `allow-same-origin` - Allow same-origin requests
- `allow-popups` - Allow popup windows
- `allow-fullscreen` - Allow fullscreen mode

### Example Sandbox Usage

```mdx
<Iframe 
  src="https://untrusted-content.example.com"
  sandbox="allow-scripts"
  title="Sandboxed external content"
/>
```

## Accessibility Features

- **Title Attribute**: Required for screen reader identification
- **Keyboard Navigation**: Embedded content maintains keyboard accessibility
- **Focus Management**: Proper focus handling when entering/exiting iframe
- **Screen Reader Support**: Announces iframe content and purpose

## Performance Considerations

- **Lazy Loading**: Iframes load only when they enter the viewport
- **Resource Impact**: External content affects page load time
- **Network Requests**: Each iframe makes additional HTTP requests
- **Memory Usage**: Embedded content consumes browser memory

## Best Practices

- Always provide descriptive titles for accessibility
- Use appropriate sandbox restrictions for security
- Consider the performance impact of embedded content
- Test embedded content across different devices and browsers
- Ensure embedded content is mobile-friendly
- Provide fallback content for when iframes fail to load

## Troubleshooting

### Content Not Loading
- Check if the source URL is accessible
- Verify CORS and X-Frame-Options headers
- Ensure the content allows embedding

### Security Errors
- Review sandbox restrictions
- Check for mixed content issues (HTTP vs HTTPS)
- Verify domain permissions
