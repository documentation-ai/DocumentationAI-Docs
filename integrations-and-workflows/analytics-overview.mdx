Track page views, user behavior, and product events by connecting your preferred analytics provider. Provide your project ID or tracking key and publish — no custom scripts required.

## Prerequisites

- An active account with the analytics provider you want to connect
- The project ID, tracking key, or measurement ID from that provider's dashboard
- Publishing access to your documentation project

## Configure from the Editor

<Steps>
  <Step title="Open Site Config" icon="settings" title-type="p">
    In the Editor, click **Site Config** in the top toolbar.
  </Step>

  <Step title="Navigate to Integrations" icon="plug" title-type="p">
    Select **Integrations** from the left sidebar. A list of supported providers appears.
  </Step>

  <Step title="Enable a provider" icon="toggle-right" title-type="p">
    Select the analytics provider you want to connect and enter the required credential. Each provider displays a label indicating the expected value (for example, "GTM Container ID" or "PostHog API Key").
  </Step>

  <Step title="Publish" icon="upload" title-type="p">
    Click **Publish** to deploy the integration to your live site.
  </Step>
</Steps>

## Configure via documentation.json

Add integrations directly to `documentation.json` if you manage configuration in version control:

```json
{
  "integrations": {
    "gtm": {
      "tagId": "GTM-XXXXXX"
    },
    "posthog": {
      "apiKey": "phc_your_project_key",
      "apiHost": "https://us.i.posthog.com"
    }
  }
}
```

## Troubleshooting

**Integration not appearing on the live site**: Verify you have published after adding the integration. Check the browser developer console for script loading errors.

**Ad blockers**: Many analytics scripts are blocked by browser ad-blocking extensions. This is expected behavior and does not indicate a configuration issue.

**Wrong tracking ID**: Double-check the exact property name and value format. For example, Google Tag Manager expects a `tagId` starting with `GTM-`, while Google Analytics expects a `measurementId` starting with `G-`.
