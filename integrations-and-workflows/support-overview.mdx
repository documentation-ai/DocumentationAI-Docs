Add live chat and support widgets to help your users get assistance without leaving your documentation. Provide your app ID and publish — no custom scripts required.

## Prerequisites

- An active account with the support provider you want to connect
- The app ID or website ID from that provider's dashboard
- Publishing access to your documentation project

## Configure from the Editor

<Steps>
  <Step title="Open Site Config" icon="settings" title-type="p">
    In the Editor, click **Site Config** in the top toolbar.
  </Step>

  <Step title="Navigate to Integrations" icon="plug" title-type="p">
    Select **Integrations** from the left sidebar.
  </Step>

  <Step title="Enable a provider" icon="toggle-right" title-type="p">
    Select the support provider you want to connect and enter the required credential (for example, "Intercom App ID" or "Crisp Website ID").
  </Step>

  <Step title="Publish" icon="upload" title-type="p">
    Click **Publish** to deploy the widget to your live site.
  </Step>
</Steps>

## Configure via documentation.json

Add support integrations directly to `documentation.json`:

```json
{
  "integrations": {
    "intercom": {
      "appId": "your_app_id"
    }
  }
}
```

## Troubleshooting

**Widget not appearing**: Verify you have published after adding the integration. Some widgets only appear on the live site, not in editor preview.

**Ad blockers**: Chat widgets may be blocked by browser ad-blocking extensions. This is expected behavior.

**Multiple chat widgets**: Only enable one support provider at a time to avoid conflicting widgets on the page.
