Documentation.AI includes built-in integrations for popular analytics and customer support tools. Provide your project ID or tracking key and publish — no custom scripts required.

## Prerequisites

- An active account with the analytics or chat provider you want to connect

- The project ID, tracking key, or app ID from that provider's dashboard

- Publishing access to your documentation project

## Configure integrations from the Editor

<Steps>
  <Step title="Open Site Config" icon="settings" title-type="p">
    In the Editor, click **Site Config** in the top toolbar.
  </Step>

  <Step title="Navigate to Integrations" icon="plug" title-type="p">
    Select **Integrations** from the left sidebar. A list of supported providers appears.
  </Step>

  <Step title="Enable an integration" icon="toggle-right" title-type="p">
    Select the provider you want to connect and enter the required credential. Each provider displays a label indicating the expected value (for example, "GTM Container ID" or "PostHog API Key").
  </Step>

  <Step title="Publish" icon="upload" title-type="p">
    Click **Publish** to deploy the integration to your live site. Integration changes appear in the **Changes** panel alongside content edits.
  </Step>
</Steps>

## Configure integrations via documentation.json

If you manage configuration in version control, add integrations directly to `documentation.json`:

```json
{
  "integrations": {
    "gtm": {
      "tagId": "GTM-XXXXXX"
    },
    "posthog": {
      "apiKey": "phc_your_project_key",
      "apiHost": "https://us.i.posthog.com"
    },
    "intercom": {
      "appId": "your_app_id"
    }
  }
}
```

Each integration is a key under the `integrations` object. Only include the providers you want to enable.

## Complete example

Here is a `documentation.json` snippet with multiple integrations enabled:

```json
{
  "integrations": {
    "gtm": { "tagId": "GTM-XXXXXX" },
    "posthog": {
      "apiKey": "phc_your_project_key",
      "apiHost": "https://us.i.posthog.com"
    },
    "plausible": { "domain": "docs.example.com" },
    "intercom": { "appId": "abc123de" }
  }
}
```

<Callout kind="tip" collapsed="false">
  You can enable multiple integrations at the same time. For example, use Google Tag Manager for marketing tags and PostHog for product analytics alongside Intercom for support chat.
</Callout>

## Troubleshooting

**Integration not appearing on the live site**: Verify you have published after adding the integration. Check the browser developer console for script loading errors.

**Ad blockers**: Many analytics and chat scripts are blocked by browser ad-blocking extensions. This is expected behavior and does not indicate a configuration issue.

**Wrong tracking ID**: Double-check the exact property name and value format. For example, Google Tag Manager expects a `tagId` starting with `GTM-`, while Google Analytics expects a `measurementId` starting with `G-`.

**Content Security Policy (CSP) errors**: If your documentation is served behind a CSP, ensure the integration provider's script domains are allowed.
